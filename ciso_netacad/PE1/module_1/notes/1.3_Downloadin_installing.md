1.3.1 Begin your Python journey
How to get Python and how to use it
There are several ways to get Python 3, depending on your operating system.

Many Linux users already have Python installed.

Some Linux tools and desktop parts also use Python.

If you're on Linux, open the terminal and type:

python3

at the shell prompt, press Enter and wait. If you see something like this:

Terminal/console screen

Sketch:

```text
+----------------------------------+
| Terminal                          |
|----------------------------------|
| $ python3                        |
| Python 3.x.x ...                 |
| >>>                              |
+----------------------------------+
```

then you are ready.

If Python 3 is not installed, use your Linux package manager to install python3.

Everyone else can download Python from https://www.python.org/downloads/.

1.3.2 How to download, install, and configure Python
The website usually detects your operating system for you.

Choose Python 3 and download the latest version.

If you're on Windows, open the .exe file and follow the installer.

Keep the default settings, but select Add Python 3.x to PATH.

This makes Python easier to run from the command line.

If you're on macOS, download the .pkg file from the Python website.

1.3.3 Starting your work with Python
Now that Python is installed, let’s check that it works.

This first test is simple, but important.

There are many ways to use Python.

To start your work, you need the following tools:

an editor for writing code;
a console for running code;
a debugger for stepping through code.
The standard Python 3 installation also includes IDLE.

IDLE means Integrated Development and Learning Environment.

Find IDLE under Python 3.x and open it. You should see a shell window like this:

Python shell

```text
|----------------------------------|

|                                  |

```

1.3.4 Your very first program before your first program...
It is now time to write your first Python 3 program. It will be very simple.

First, create a new source file. In IDLE, choose File and then New file.

Python 3.7.0 Shell – A New File

Sketch:

```text
+----------------------------------+
| IDLE - New File                  |
|----------------------------------|
| File  Edit  Format  Run  Help    |
|                                  |
| print("Hisssssss...")            |
+----------------------------------+
```

IDLE opens a new editor window where you write code.

This is the editor window. It is different from the shell window.

It is best to name the file before you start.

Click File, then Save as..., choose a folder, and name the file.

Python 3.7.0 Shell – Saving a new file named "snake.py"

Note: let the file keep the .py extension.

Now type one line in the editor:

The line looks like this:

print("Hisssssss...")

You can copy and paste it.

We will explain it later.

Use straight quotation marks, not curved ones.

1.3.5 How to spoil and fix your code
Now open IDLE again.

Open the file you saved earlier and press F5 to run it again.

IDLE has one helpful feature.

Delete the closing parenthesis and type it again.
Your code should look like this:

Output
Hisssssss...
IDLE highlights matching parentheses. This helps you keep them in pairs.

Python 3.7.0 IDLE – Running a program after removing parenthesis from the print() function

Sketch:

```text
+-------------------------------------------+
| IDLE Editor                               |
|-------------------------------------------|
| print("Hisssssss..."                      |
|                 ^                         |
|  matching parenthesis highlight           |
+-------------------------------------------+
```

Now remove the closing parenthesis again. The code has a syntax error, so IDLE should stop you.

Try to run it again and save the file when IDLE asks.

Watch the windows carefully.

A new window appears with an EOF error.

The editor also shows where the error is.

Python 3.7.0 IDLE – Console message after running a program with removed parenthesis from the print() function

Sketch:

```text
+-------------------------------------------+
| Python Shell                              |
|-------------------------------------------|
| Traceback (most recent call last):        |
|   ... EOF while scanning ...              |
+-------------------------------------------+
```

Fix the code:

print("Hisssssss...")

Run it again.

Now remove one letter from print and run it again.

Python 3.7.0 IDLE – Removing the letter "t" from the print() function

Sketch:

```text
+-------------------------------------------+
| IDLE Editor                               |
|-------------------------------------------|
| prin("Hisssssss...")                      |
|   ^ missing letter                        |
+-------------------------------------------+
```

This error looks different.

Python 3.7.0 IDLE - Console error message after running a program with the "t" letter removed from the print() function

Sketch:

```text
+-------------------------------------------+
| Python Shell                              |
|-------------------------------------------|
| NameError: name 'prin' is not defined     |
+-------------------------------------------+
```

It is a different kind of error.

The editor window will not help much, but the console will.

The red message shows:

the traceback;
the location of the error;
the line with the problem;
the error name and a short explanation.

Try making a new file and printing a different word, such as roar!, meow, or oink!. Then break and fix the code.
